  <action>
1. Initialize git if not already initialized, and ensure all Phase 4 changes are committed:
   ```bash
   git init  # if needed
   git add -A
   git commit -m "feat: production polish -- performance, semantic HTML, OG tags, responsive fixes"
   ```

2. Create a public GitHub repository using the GitHub CLI:
   ```bash
   gh repo create jack-engineering-portfolio --public --source=. --push
   ```
   Per user decision: repo name is "jack-engineering-portfolio", public visibility (required for Vercel free tier).

3. Deploy to Vercel using the Vercel CLI:
   ```bash
   npx vercel --yes
   ```
   Vercel auto-detects the Vite framework, uses `tsc -b && vite build` as build command, and `dist/` as output directory. No vercel.json needed.

   If the Vercel CLI prompts for login/authentication, this becomes a checkpoint for the user to authenticate. After auth, re-run the deploy command.

4. After successful deploy, note the production URL (likely `jack-engineering-portfolio.vercel.app` or similar).

5. Link the Vercel project to the GitHub repo for auto-deploy on push:
   ```bash
   npx vercel link
   npx vercel git connect
   ```

6. Set up production deployment (so pushes to main auto-deploy):
   ```bash
   npx vercel --prod --yes
   ```

7. Update OG meta tag URLs in `index.html` to use the actual Vercel domain:
   - Replace `https://jack-engineering-portfolio.vercel.app` with the actual production URL in:
     - `og:url`
     - `og:image`
     - `twitter:image`
   - Only update if the actual domain differs from the placeholder.

8. If URLs were updated, commit and push to trigger auto-deploy:
   ```bash
   git add index.html
   git commit -m "fix: update OG meta URLs to actual Vercel domain"
   git push
   ```
