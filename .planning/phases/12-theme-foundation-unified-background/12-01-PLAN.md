  <files>index.html, src/main.tsx, package.json</files>

1. **Add blocking script to index.html** in the `<head>` section, BEFORE the closing `</head>` tag and BEFORE any `<link>` or module scripts. Place it right after the `<title>` tag:

```html
<script>
  // Synchronous dark mode detection -- prevents FOUT
  (function() {
    var d = document.documentElement;
    var prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
    if (prefersDark) d.classList.add('dark');
    d.style.colorScheme = prefersDark ? 'dark' : 'light';
    // Suppress transitions on initial load
    d.classList.add('no-transition');
    requestAnimationFrame(function() {
      requestAnimationFrame(function() {
        d.classList.remove('no-transition');
      });
    });
  })();
</script>
```

This script runs synchronously before any CSS or JS modules load, applying .dark class and color-scheme immediately. The double-rAF pattern removes no-transition after the first paint cycle so subsequent theme changes will animate.

2. **Add matchMedia listener to src/main.tsx** after the `document.body.classList.add('hydrated')` line:

```typescript
// Listen for system preference changes and update .dark class + color-scheme
const darkMql = window.matchMedia('(prefers-color-scheme: dark)');
function applySystemTheme(e: MediaQueryListEvent | MediaQueryList) {
  const root = document.documentElement;
  root.classList.toggle('dark', e.matches);
  root.style.colorScheme = e.matches ? 'dark' : 'light';
}
darkMql.addEventListener('change', applySystemTheme);
```

This keeps the .dark class in sync with system preference changes at runtime (e.g., user toggles OS dark mode while the site is open).

3. **Remove next-themes** package:
   Run `npm uninstall next-themes` to remove the unused Next.js-specific dependency.
