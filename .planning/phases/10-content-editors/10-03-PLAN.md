  **Step 1 — Create `src/admin/editors/PapersEditor.tsx`:**
  List-type editor for Paper[]:
  - Uses `useContentEditor<Paper[]>({ contentType: 'papers', schema: z.array(paperSchema), onDirtyChange, saveRef })`
  - Local `activeIndex` state.
  - ItemList at top: `getLabel={(item) => item.title || 'Untitled Paper'}`.
  - Form for active item:
    - `<SectionHeader>Details</SectionHeader>`
    - FormField for `id` — **read-only** (auto-derived from title). Show as a disabled/grayed text showing the current ID value. When `title` changes, auto-update `id` via `toKebabCase(title)` (import from `src/admin/upload.ts`). Only auto-generate for new items (items where `id` was empty or just created). For existing items that already have an `id`, do NOT auto-update it (changing the ID of an existing item would break references).
    - FormField for `title`.
    - FormField for `descriptor` with `multiline`.
    - `<SectionHeader>PDF</SectionHeader>`
    - UploadZone for `pdfPath`: `accept={['.pdf']}`, `maxSize={10 * 1024 * 1024}`, `context={{ contentType: 'papers', field: 'pdfPath', itemId: activeItem.id }}`, `currentFile={activeItem.pdfPath}`, `onUploaded={(path) => { update pdfPath on the active item }}`
  - Add new paper: append `{ id: '', title: '', descriptor: '', pdfPath: '' }`, set activeIndex to new last.
  - Delete with `window.confirm('Delete this paper?')`.

  **Step 2 — Create `src/admin/editors/ProjectsEditor.tsx`:**
  The most complex editor. List-type for Project[]:
  - Uses `useContentEditor<Project[]>({ contentType: 'projects', schema: z.array(projectSchema), onDirtyChange, saveRef })`
  - Local `activeIndex` state.
  - ItemList at top: `getLabel={(item) => item.title || 'Untitled Project'}`.
  - Form for active item, grouped with section headers:

    **`<SectionHeader>Details</SectionHeader>`**
    - FormField for `id` — same auto-derive pattern as Papers (auto from title for new items, read-only for existing).
    - FormField for `title`.
    - FormField for `brief`.
    - FormField for `summary` with `multiline`.
    - FormField for `domain`.
    - Checkbox for `featured` — use shadcn `Checkbox` component. Label: "Featured project". Value: `activeItem.featured`. onChange: update the active item's `featured` boolean.

    **`<SectionHeader>Media</SectionHeader>`**
    - UploadZone for `thumbnail`: `accept={['.jpg', '.jpeg', '.png', '.svg', '.webp']}`, `maxSize={10 * 1024 * 1024}`, `context={{ contentType: 'projects', field: 'thumbnail', itemId: activeItem.id }}`, `currentFile={activeItem.thumbnail}`, `onUploaded` updates thumbnail.
    - For `images` array: render each existing image path as a small preview card with a remove (X) button, plus an UploadZone at the end labeled "Add Image" that appends the new path. `context={{ contentType: 'projects', field: 'images', itemId: activeItem.id }}`. When `onUploaded` fires, append the path to the `images` array.

    **`<SectionHeader>Tech & Links</SectionHeader>`**
    - TagInput for `techStack`: `label="Tech Stack"`, `tags={activeItem.techStack}`, onChange updates techStack array.
    - StructuredArrayField for `links`: `label="Links"`, `fields={[{ key: 'label', label: 'Label' }, { key: 'url', label: 'URL' }]}`, items and onChange wired to the active project's links array.

  - Add new project: append `{ id: '', title: '', brief: '', summary: '', thumbnail: '', images: [], domain: '', techStack: [], links: [], featured: false }`, set activeIndex to new last.
  - Delete with `window.confirm('Delete this project?')`.

  **Common patterns (both editors):**
  - Props: `{ onDirtyChange, saveRef }` (same as all other editors).
  - Loading skeleton while data loads.
  - All data via API, never direct imports.
  - Field updates use `setData` to clone array, modify active item, return new array. Always call `onDirtyChange(true)`.
  - Pass relevant `fieldErrors` to components. For array items, handle nested error display reasonably (top-level array error alongside the item form).
