<objective>
Fix the HMR suppression path normalization bug that prevents handleHotUpdate from matching admin-written files on Windows.

Purpose: On Windows, path.resolve() produces backslash-separated paths (e.g., C:\...\src\data\hero.ts) while Vite's handleHotUpdate delivers forward-slash normalized paths (e.g., C:/.../src/data/hero.ts). The Set.has() comparison never matches, meaning HMR suppression is a no-op on Windows. This closes the single remaining gap from Phase 8 verification (INFRA-05 partial).

Output: Patched vite-plugin-admin-api.ts with normalizePath applied to stored paths, plus a targeted unit test.
</objective>
