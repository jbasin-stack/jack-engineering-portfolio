Modify `src/components/sections/Expertise.tsx` to add a directional sliding transition when switching tabs. The content should slide in from the right when navigating to a higher-index tab and slide in from the left when navigating to a lower-index tab, while preserving the existing blur/scale/opacity transition.

**Step 1: Track tab direction**

Add a `useRef` to track the previous tab index, and compute a `direction` value (+1 or -1):

```tsx
import { useState, useRef } from 'react';

// Inside the component, after the tabs array derivation:
const directionRef = useRef(0);

// Replace the simple setActiveTab with a direction-aware handler:
function handleTabChange(newTabId: string) {
  const oldIndex = tabs.findIndex((t) => t.id === activeTab);
  const newIndex = tabs.findIndex((t) => t.id === newTabId);
  directionRef.current = newIndex > oldIndex ? 1 : -1;
  setActiveTab(newTabId);
}
```

Pass `handleTabChange` to `AnimatedTabs onChange` instead of the bare `setActiveTab`.

**Step 2: Create direction-aware variants**

Define variants as functions that receive a `direction` number via AnimatePresence `custom` prop:

```tsx
const slideVariants = {
  initial: (direction: number) => ({
    x: direction * 60,
    opacity: 0,
    scale: 0.96,
    filter: 'blur(8px)',
  }),
  animate: {
    x: 0,
    opacity: 1,
    scale: 1,
    filter: 'blur(0px)',
  },
  exit: (direction: number) => ({
    x: direction * -60,
    opacity: 0,
    scale: 0.96,
    filter: 'blur(8px)',
  }),
};
```

The `x: direction * 60` means moving right (+60px) for forward navigation and left (-60px) for backward navigation on initial. The exit is reversed so the outgoing panel slides away in the opposite direction.

Use 60px as the slide distance -- enough to be perceptible without being dramatic. This is a subtle directional cue, not a full-screen slide.

**Step 3: Wire AnimatePresence custom prop**

Update the AnimatePresence block:

```tsx
<AnimatePresence mode="wait" custom={directionRef.current}>
  <motion.div
    key={activeTab}
    custom={directionRef.current}
    role="tabpanel"
    id={`panel-${activeTab}`}
    variants={slideVariants}
    initial="initial"
    animate="animate"
    exit="exit"
    transition={{ duration: 0.25, ease: easing.out }}
    className="mt-6 min-h-[200px] rounded-xl backdrop-blur-md bg-white/10 dark:bg-white/5 border border-white/20 dark:border-white/10 p-6"
  >
```

IMPORTANT: Both `AnimatePresence` and `motion.div` need the `custom` prop. AnimatePresence needs it so the exiting element gets the updated direction value (since it is unmounting, it reads `custom` from the parent).

**Step 4: Verify no regressions**

The glassmorphic panel classes, two-column grid, and min-height must remain unchanged. The only modification is adding the `x` transform to the existing blur/scale/opacity animation and swapping from inline variant objects to the `slideVariants` object.

Ensure `overflow-hidden` is NOT added to the panel container -- the slight slide offset should be visible, not clipped.
