  <name>Task 1: Create PdfViewer component with react-pdf, page controls, zoom, and download</name>
  <files>src/components/pdf/PdfViewer.tsx</files>
  <action>
Create `src/components/pdf/PdfViewer.tsx` -- the shared PDF viewer used by Papers section and Contact resume button.

**Props interface:**
```typescript
interface PdfViewerProps {
  file: string;           // Path to PDF (e.g. '/papers/paper.pdf')
  title: string;          // Display title for Dialog/Drawer header
  open: boolean;
  onOpenChange: (open: boolean) => void;
}
```

**Worker configuration (CRITICAL -- must be in same file as Document):**
```typescript
import { Document, Page, pdfjs } from 'react-pdf';
import 'react-pdf/dist/Page/AnnotationLayer.css';
import 'react-pdf/dist/Page/TextLayer.css';

pdfjs.GlobalWorkerOptions.workerSrc = '/pdf.worker.min.mjs';
```

**State:**
- `numPages: number` (0 initially, set by onLoadSuccess)
- `pageNumber: number` (1 initially)
- `scale: number` (1.0 initially)

**Controls toolbar:**
Use lucide-react icons (already installed): `ChevronLeft`, `ChevronRight`, `ZoomIn`, `ZoomOut`, `Download`, `X`.

Toolbar layout: `flex items-center justify-between` with:
- Left group: page nav -- prev button (disabled if page 1), page indicator text (`{pageNumber} / {numPages}`), next button (disabled if last page)
- Center/right group: zoom out button, zoom level text (e.g. "100%"), zoom in button
- Far right: download button (anchor tag `<a href={file} download>` wrapping Download icon), close button

Zoom step: 0.25 increments. Min 0.5, max 2.5.

Button styling: `rounded-lg p-2 hover:bg-silicon-50 text-silicon-600 transition-colors disabled:opacity-30 disabled:cursor-not-allowed`

**PDF display area:**
- Scrollable container below toolbar: `overflow-auto flex-1` with `flex justify-center`
- `<Document file={file} onLoadSuccess={({ numPages }) => setNumPages(numPages)}>`
- `<Page pageNumber={pageNumber} scale={scale} />`
- Handle loading state: show a "Loading..." text centered
- Handle error state: show "Failed to load PDF" with a download fallback link

**Responsive Dialog/Drawer pattern:**
- Use `useIsMobile()` hook
- Desktop: `Dialog` with `DialogContent className="max-w-[90vw] h-[90vh] flex flex-col p-0"` -- near-full-screen per CONTEXT.md. Toolbar at top, PDF display fills remaining space.
- Mobile: `Drawer` with `DrawerContent className="h-[95vh] flex flex-col p-0"` -- full-height per CONTEXT.md. Same internal layout.
- Both: `DialogTitle`/`DrawerTitle` in header shows the `title` prop

**Lenis scroll lock:**
When the viewer opens, Lenis must be stopped so the PDF content can scroll independently. When it closes, Lenis must restart. Use the same pattern as the mobile nav menu -- access the Lenis instance and call `lenis.stop()` / `lenis.start()`. Use a `useEffect` that watches the `open` prop:
```typescript
useEffect(() => {
  const lenis = (window as any).__lenis;  // or however the project exposes it
  if (open) lenis?.stop();
  else lenis?.start();
  return () => lenis?.start(); // cleanup
}, [open]);
```
Check SmoothScroll.tsx for how Lenis instance is exposed. If it's on `window.__lenis` or via a ref/context, use that. If not exposed, find the pattern from the mobile nav implementation.

**Key constraints:**
- No spring animations anywhere
- Download button must work even if PDF fails to render (pure `<a href download>`)
- Worker path is `/pdf.worker.min.mjs` (stable, in public/)

**PaperRow.tsx:**

Create a clean row component for individual paper display.

Props:
```typescript
interface PaperRowProps {
  paper: Paper;
  onView: () => void;
}
```

Layout: horizontal flex row with baseline alignment (`flex items-baseline justify-between`).
- Left side: paper title (`font-semibold text-ink`) + descriptor (`text-sm text-silicon-600 ml-2` or on second line)
- Right side: "View" button styled as an accent text link/button (`text-accent font-medium hover:underline cursor-pointer`)
- Separator between rows: `border-b border-silicon-200/30` on each row (or use `divide-y` on parent)
- Padding: `py-4` per row

Wrap in `motion.div` with `variants={fadeUpVariant}` for staggered entry with section.

**PapersSection.tsx:**

Create the Papers section wrapper.

- Follow established section pattern: `motion.section` with `id="papers"`, `aria-label="Papers"`, `className="px-6 py-24"`, `variants={sectionVariants}`, `initial="hidden"`, `whileInView="visible"`, `viewport={{ once: true, amount: 0.15 }}`
- Container: `mx-auto max-w-5xl`
- Section heading: `motion.h2` with `variants={fadeUpVariant}`, text "Papers"
- State: `const [viewingPdf, setViewingPdf] = useState<{ file: string; title: string } | null>(null)`
- Map over `papers` data, rendering `PaperRow` for each:
  - `key={paper.id}`
  - `paper={paper}`
  - `onView={() => setViewingPdf({ file: paper.pdfPath, title: paper.title })}`
- Wrap rows in a `motion.div` with `variants={fadeUpVariant}` and `className="mt-12 divide-y divide-silicon-200/30"`
- Render `PdfViewer` at the bottom:
  ```tsx
  <PdfViewer
    file={viewingPdf?.file ?? ''}
    title={viewingPdf?.title ?? ''}
    open={viewingPdf !== null}
    onOpenChange={(open) => { if (!open) setViewingPdf(null); }}
  />
  ```

**Design intent from CONTEXT.md:** Papers as clean rows (not cards) deliberately contrasts with the visual bento grid -- signals "documents to read" vs "projects to explore". Keep it minimal and typographic.

The Contact section's resume download button can be updated to also open the PDF viewer. However, since Contact.tsx was built in Phase 2, wiring the PdfViewer to the resume button will happen in Plan 04 (wiring plan) to avoid modifying Contact.tsx here.
