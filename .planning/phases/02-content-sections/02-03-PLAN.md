  <action>
    Build the scroll-driven timeline component per CONTEXT.md locked decisions. This is the most complex component in Phase 2.

    **Overall structure:**
    ```
    <section id="timeline" className="px-6 py-24">
      <div className="mx-auto max-w-5xl">
        <h2>Timeline</h2>
        <div ref={containerRef} className="relative mt-12 pl-8">
          <!-- Progressive fill line (absolute positioned on the left) -->
          <motion.div style={{ scaleY: scrollYProgress }} />
          <!-- Milestone nodes -->
          {milestones.map(milestone => <TimelineNode />)}
        </div>
      </div>
    </section>
    ```

    **Scroll-driven fill implementation:**
    1. Use `useRef<HTMLDivElement>(null)` for the container ref
    2. Call `useScroll({ target: containerRef, offset: ['start 0.8', 'end 0.6'] })` to get `scrollYProgress`
       - "start 0.8" = fill begins when container top is at 80% of viewport
       - "end 0.6" = fill completes when container bottom is at 60% of viewport
    3. The fill line is a `<motion.div>` with:
       - `className="absolute left-0 top-0 h-full w-0.5 origin-top bg-accent"`
       - `style={{ scaleY: scrollYProgress }}`
       - This creates the "invisible until filled" effect: the line IS the fill, there is no grey track underneath. The 2px width (`w-0.5` = 2px) per CONTEXT decision.

    **CRITICAL PITFALL -- MotionValue in conditional rendering:**
    Per RESEARCH.md Pitfall 1, `useTransform` returns a MotionValue, NOT a boolean. You CANNOT use it in className ternaries or conditional rendering directly. Instead:

    ```typescript
    // For each TimelineNode:
    const [isActive, setIsActive] = useState(false);
    useMotionValueEvent(scrollProgress, 'change', (latest) => {
      setIsActive(latest >= threshold);
    });
    ```

    **TimelineNode sub-component (inline in same file):**
    Each node receives: `milestone`, `index`, `total`, `scrollProgress` (MotionValue).

    ```typescript
    function TimelineNode({ milestone, index, total, scrollProgress }: {
      milestone: TimelineMilestone;
      index: number;
      total: number;
      scrollProgress: MotionValue<number>;
    }) {
      const threshold = index / Math.max(total - 1, 1);
      const [isActive, setIsActive] = useState(false);

      useMotionValueEvent(scrollProgress, 'change', (latest) => {
        setIsActive(latest >= threshold);
      });

      return (
        <div className="relative pb-12 last:pb-0">
          {/* Node dot -- hollow grey until active, then filled accent */}
          <div className={`absolute -left-8 top-1 h-3 w-3 rounded-full border-2 transition-colors duration-300
            ${isActive ? 'border-accent bg-accent' : 'border-silicon-200 bg-cleanroom'}`}
          />
          {/* Milestone content -- fades in when active */}
          <div className={`transition-all duration-500
            ${isActive ? 'translate-y-0 opacity-100' : 'translate-y-2 opacity-0'}`}>
            <span className="text-sm text-silicon-400">{milestone.date}</span>
            <h3 className="mt-1 text-lg font-semibold text-ink">{milestone.title}</h3>
            <p className="mt-1 text-silicon-600">{milestone.description}</p>
          </div>
        </div>
      );
    }
    ```

    **Key implementation details:**
    - Import `useRef, useState` from `react`
    - Import `motion, useScroll, useMotionValueEvent, type MotionValue` from `motion/react`
    - Import `milestones` from `../../data/timeline`
    - Section heading uses same style as other sections: `text-2xl font-bold text-ink`
    - Timeline line: 2px weight (`w-0.5`) per CONTEXT (thicker than 0.5px border system for visual presence)
    - Node dots: `h-3 w-3` (12px) with `border-2` -- start hollow grey, transition to filled accent
    - Milestone content: three tiers (date label, bold title, one-line description) per CONTEXT
    - Content fades up when active using CSS transition (NOT Motion whileInView, because activation is scroll-progress driven, not viewport driven)
    - The fill line has NO grey track behind it. The `bg-accent` line scales from 0 to 1 against a transparent background. This creates the "invisible until filled" effect per CONTEXT.
    - `id="timeline"` for anchoring but NO nav link needed (per RESEARCH.md Open Question 1)
    - Node spacing: `pb-12` between nodes, `last:pb-0` for the last one
    - The `pl-8` on the container provides space for the line and dots on the left
