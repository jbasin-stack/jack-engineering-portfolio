# Phase 14.1: Timeline Overhaul - Research

**Researched:** 2026-03-27
**Domain:** Editorial timeline layout, scroll-triggered animation, CSS layout variants
**Confidence:** HIGH

## Summary

Phase 14.1 replaces the current scroll-drawn SVG timeline with an editorial-style timeline inspired by ranomi.nl/highlights. The current implementation uses a thin SVG path with small node markers and pulse-ring animations -- it feels cramped and dated compared to the recently rebuilt Expertise tabs and Project carousel. The replacement uses large year anchors, varied image layout modes (large/half/overlay), generous spacing, a thin accent connector line with dot markers, and `whileInView` fade-in entry animations.

The project already has all required dependencies installed: `motion/react` v12.38+ for `whileInView` animations, Tailwind CSS v4 for utility styling, and the oklch color system for accent theming. No new npm packages are needed. The existing `sectionVariants` and `fadeUpVariant` from `styles/motion.ts` provide the exact animation pattern needed. The data model (`TimelineMilestone` in `types/data.ts`) needs only a single optional `image?: string` field added.

**Primary recommendation:** Complete replacement of `Timeline.tsx` with an editorial layout component that cycles through three layout variants per entry, uses `whileInView` with staggered children for scroll-triggered fade-in, and maintains a thin left-edge accent line with animated dot markers.

### Pattern 1: Editorial Layout Variant Cycling
**What:** Each timeline entry is assigned one of three layout variants in a repeating pattern based on its array index.
**When to use:** When entries need visual variety without manual per-entry configuration.

```typescript
// Layout variants cycle: large -> half -> overlay -> large -> ...
type LayoutVariant = 'large' | 'half' | 'overlay';
const LAYOUT_CYCLE: LayoutVariant[] = ['large', 'half', 'overlay'];

function getLayoutVariant(index: number): LayoutVariant {
  return LAYOUT_CYCLE[index % LAYOUT_CYCLE.length];
}
```

**Layout variant visual behavior:**

| Variant | With Image | Without Image (text-only) |
|---------|-----------|--------------------------|
| `large` | Full-width image above text content | Text content spans full width, year is extra large |
| `half` | Image and text side-by-side (50/50 grid) | Text takes full width, extra breathing room |
| `overlay` | Text overlaid on image with gradient scrim | Same as `large` text-only -- no visual difference needed |

### Pattern 2: whileInView with Staggered Children (Existing Pattern)
**What:** Parent container uses `whileInView` to trigger variant propagation; children stagger in sequentially.
**When to use:** Each timeline entry's sub-elements (year, title, description, image, dot) animate in together.

```typescript
// Source: Existing pattern from WhoAmI.tsx and Expertise.tsx
import { sectionVariants, fadeUpVariant } from '../../styles/motion';

// Each entry is a motion.div with whileInView
<motion.article
  variants={entryVariants}
  initial="hidden"
  whileInView="visible"
  viewport={{ once: true, amount: 0.2 }}
>
  <motion.div variants={fadeUpVariant}>
    {/* Year */}
  </motion.div>
  <motion.div variants={fadeUpVariant}>
    {/* Title + description */}
  </motion.div>
  <motion.div variants={fadeUpVariant}>
    {/* Image (if present) */}
  </motion.div>
</motion.article>
```

### Pattern 3: Dot Marker Animation Synchronized with Entry
**What:** The dot on the left connector line animates in (scale + opacity) at the same time as its parent entry.
**When to use:** Visual anchoring of the connector line to entry positions.

```typescript
// Dot uses same variant system as entry content
const dotVariant: Variants = {
  hidden: { opacity: 0, scale: 0 },
  visible: {
    opacity: 1,
    scale: 1,
    transition: { duration: 0.4, ease: easing.out },
  },
};
```

### Pattern 4: Year Extraction from Date String
**What:** Parse the 4-digit year from the existing date field format ("Sep 2023" -> "2023").
**When to use:** Displaying the large accent year without changing the data model.

```typescript
function extractYear(date: string): string {
  const match = date.match(/\d{4}/);
  return match ? match[0] : date;
}
```

### Anti-Patterns to Avoid
- **Don't use useScroll + useTransform for this:** The old timeline used scroll-bound motion values for progressive drawing. The new design uses discrete viewport entry triggers (`whileInView`), which is simpler and more performant.
- **Don't use one-shot useState activation pattern:** The old component tracked `hasActivated` state per node with `useMotionValueEvent`. The new approach uses `whileInView` with `viewport={{ once: true }}` which handles this natively.
- **Don't add layout variant data to the data model:** Variants are computed from array index, not stored. This keeps data clean and allows layout changes without data migration.
- **Don't create separate component files for each layout variant:** The three variants differ only in CSS classes and grid structure -- use conditional classes within a single `TimelineEntry` subcomponent.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Scroll-triggered animation | Custom IntersectionObserver + state | Motion `whileInView` + variants | Already used in WhoAmI, Expertise; handles reduced motion via `MotionConfig` |
| Staggered children animation | Manual delay calculation | `sectionVariants` with `staggerChildren` | Existing pattern in `styles/motion.ts` |
| Responsive grid layouts | Custom media query JS | Tailwind responsive utilities (`md:grid-cols-2`) | Consistent with entire codebase |
| Image lazy loading | Custom intersection logic | Native `loading="lazy"` attribute | Established pattern in this project |

**Key insight:** This phase requires zero new animation techniques. Every scroll and animation pattern is already established in the codebase. The work is purely layout and styling.

## Common Pitfalls

### Pitfall 1: Forgetting to Remove pulse-ring Keyframe
**What goes wrong:** Dead CSS left in app.css after the only consumer (old Timeline) is removed.
**Why it happens:** The keyframe is in global CSS, easy to miss when replacing the component.
**How to avoid:** Explicitly remove the `@keyframes pulse-ring` block (lines 282-291 in `src/styles/app.css`) and verify no other components reference `pulse-ring`.
**Warning signs:** Grep for `pulse-ring` across codebase should return zero results after cleanup.

### Pitfall 2: Year Extraction Failing on Edge-Case Date Formats
**What goes wrong:** If a date like "Fall 2025" or "2023" (no month) is in the data, the regex might behave unexpectedly.
**Why it happens:** Current data uses "Mon YYYY" format but admin editor accepts freeform text.
**How to avoid:** Use a robust regex `/\d{4}/` which matches any 4-digit sequence. Fallback to full date string if no match.
**Warning signs:** Test with current 8 milestone dates -- all are "Mon YYYY" format, all will match.

### Pitfall 3: Connector Line Height Misalignment
**What goes wrong:** The thin left-edge line doesn't span the full height of the timeline section, leaving gaps or overflows.
**Why it happens:** Absolute positioning with percentage heights can break with dynamic content.
**How to avoid:** Use `absolute top-0 bottom-0` (or `h-full`) on the line element within a `relative` parent that wraps all entries. The line is a simple CSS pseudo-element or a `div`, not SVG.
**Warning signs:** Line should start at first entry dot and end at last entry dot.

### Pitfall 4: Layout Variant Overflow on Mobile
**What goes wrong:** The `half` variant (side-by-side grid) breaks on narrow viewports.
**Why it happens:** Fixed two-column layout without responsive breakpoint.
**How to avoid:** Use `grid-cols-1 md:grid-cols-2` so `half` variant stacks to single column on mobile.
**Warning signs:** Test at 375px viewport width -- all variants should stack vertically.

### Pitfall 5: Overlay Variant Accessibility (Text on Image)
**What goes wrong:** Text becomes unreadable when placed over a light or busy image.
**Why it happens:** No gradient scrim or contrast layer between image and text.
**How to avoid:** Use a dark-to-transparent gradient overlay (`bg-gradient-to-t from-black/60 to-transparent`) on the overlay variant. Since shipping with no images initially, this is forward-looking but should be implemented now.
**Warning signs:** Run a contrast check on any future images added.

### Pitfall 6: Admin Schema Out of Sync with Type
**What goes wrong:** Adding `image?: string` to the TypeScript type but forgetting the Zod schema, or vice versa.
**Why it happens:** Type (`data.ts`) and schema (`admin/schemas.ts`) are separate files.
**How to avoid:** Update both in the same task. The schema change is `image: z.string().optional()`.
**Warning signs:** Admin editor will throw validation errors if schema doesn't accept the new field.

## Code Examples

### Timeline Entry Component (Recommended Structure)
```typescript
// Source: Synthesized from existing patterns in WhoAmI.tsx, Expertise.tsx, motion.ts
import { motion, type Variants } from 'motion/react';
import { easing } from '../../styles/motion';

type LayoutVariant = 'large' | 'half' | 'overlay';
const LAYOUT_CYCLE: LayoutVariant[] = ['large', 'half', 'overlay'];

// Entry container variant -- staggers children
const entryVariants: Variants = {
  hidden: {},
  visible: {
    transition: { staggerChildren: 0.12 },
  },
};

// Child fade-up (reuse existing or define inline)
const fadeUp: Variants = {
  hidden: { opacity: 0, y: 24 },
  visible: {
    opacity: 1,
    y: 0,
    transition: { duration: 0.5, ease: easing.out },
  },
};

// Dot scale-in
const dotVariant: Variants = {
  hidden: { opacity: 0, scale: 0 },
  visible: {
    opacity: 1,
    scale: 1,
    transition: { duration: 0.4, ease: easing.out },
  },
};

function TimelineEntry({
  milestone,
  index,
}: {
  milestone: TimelineMilestone;
  index: number;
}) {
  const variant = LAYOUT_CYCLE[index % LAYOUT_CYCLE.length];
  const year = milestone.date.match(/\d{4}/)?.[0] ?? milestone.date;

  return (
    <motion.article
      className="relative py-16 pl-12 md:py-20"
      variants={entryVariants}
      initial="hidden"
      whileInView="visible"
      viewport={{ once: true, amount: 0.2 }}
    >
      {/* Dot marker on the line */}
      <motion.div
        className="absolute left-0 top-20 h-[6px] w-[6px] -translate-x-1/2 rounded-full bg-accent"
        variants={dotVariant}
      />

      {/* Year anchor */}
      <motion.span
        className="text-5xl font-bold text-accent md:text-6xl"
        variants={fadeUp}
      >
        {year}
      </motion.span>

      {/* Title */}
      <motion.h3
        className="mt-3 text-xl font-semibold text-ink md:text-2xl"
        variants={fadeUp}
      >
        {milestone.title}
      </motion.h3>

      {/* Description */}
      <motion.p
        className="mt-2 max-w-xl text-base leading-relaxed text-silicon-600 md:text-lg"
        variants={fadeUp}
      >
        {milestone.description}
      </motion.p>

      {/* Image (conditional, layout-variant-dependent) */}
      {milestone.image && (
        <motion.div
          className={clsx('mt-6', {
            'w-full': variant === 'large',
            'md:w-1/2': variant === 'half',
            'relative': variant === 'overlay',
          })}
          variants={fadeUp}
        >
          <img
            src={milestone.image}
            alt=""
            loading="lazy"
            className="rounded-lg object-cover"
          />
        </motion.div>
      )}
    </motion.article>
  );
}
```

## State of the Art

| Old Approach (Current) | New Approach (This Phase) | Why Change |
|------------------------|--------------------------|------------|
| SVG `motion.path` with `pathLength` scroll binding | Static CSS line + animated dot markers | Simpler, more reliable, editorial feel |
| `useScroll` + `useTransform` per node | `whileInView` with `viewport={{ once: true }}` | Declarative, no manual threshold math |
| Small 12px node circles with border transition | 4-6px filled dots that scale in | Subtler, not the focal point |
| `pulse-ring` keyframe one-shot animation | No pulse animation | Cleaner, less busy |
| Cramped `pb-12` spacing per entry | `py-16` to `py-20` generous spacing | Breathing room, editorial feel |
| All entries identical layout | 3 cycling layout variants (large/half/overlay) | Visual rhythm, no two adjacent entries look the same |
| Text-only entries (date, title, description) | Year anchor + title + description + optional image | Richer visual hierarchy |

**Deprecated/outdated:**
- `pulse-ring` keyframe: Remove from app.css (only used by old Timeline)
- `useMotionValueEvent` threshold activation pattern: Replaced by `whileInView`
- SVG timeline gradient and path: Entirely removed

## Open Questions

1. **Overlay variant visual design (Claude's discretion)**
   - What we know: User wants an "overlay" treatment inspired by ranomi.nl's `-overlay` class
   - What's unclear: Whether text sits on top of the image with a gradient scrim, or if it's a side-by-side with image partially behind text
   - Recommendation: Implement as text overlaid on bottom portion of image with a `bg-gradient-to-t from-black/60` scrim for contrast. This is the most common editorial pattern and degrades gracefully when no image is present (falls back to large text-only).

2. **Image aspect ratios per variant (Claude's discretion)**
   - What we know: Three layout variants need different image presentations
   - What's unclear: Exact aspect ratios
   - Recommendation: `large` = 16:9 full-width, `half` = 4:3 in the half-width column, `overlay` = 3:2 as background. All use `object-cover` for consistent fill. Since shipping without images initially, these can be tuned later.

3. **Mobile behavior for half variant (Claude's discretion)**
   - What we know: Side-by-side layout breaks on mobile
   - Recommendation: Stack to single column below `md` breakpoint. Image appears above text content when stacked.
