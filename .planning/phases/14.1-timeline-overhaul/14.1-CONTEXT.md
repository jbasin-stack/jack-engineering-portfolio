# Phase 14.1: Timeline Overhaul - Context

**Gathered:** 2026-03-27
**Status:** Ready for planning

<domain>
## Phase Boundary

Complete rebuild of the Timeline section. The current implementation (small nodes on a thin SVG path with pulse-ring animations) is cramped, visually broken, and doesn't match the premium feel of the rest of the site. Replace with an editorial-style timeline inspired by ranomi.nl/highlights — large year anchors, varied image layouts, scroll-triggered fade-in entries, and a thin accent connector line with dot markers.

</domain>

<decisions>
## Implementation Decisions

### Overall vibe & approach
- Inspired by ranomi.nl/highlights editorial timeline style
- "Compact but elevated" — still one section on a single-page portfolio, not full-page-per-entry
- Drop the SVG path drawing animation entirely — fade-in entries instead
- Drop the pulse-ring keyframe animation entirely
- No progress indicator (no floating ring, no filling line)
- Motivation: current timeline has issues across visual quality, animation jank, layout, and overall concept — full rethink needed

### Entry structure
- Each entry: large accent year + bold title + description paragraph + optional image
- Varied image layout per entry cycling through 3 modes (inspired by ranomi.nl `-large`, `-halfes`, `-overlay` classes): large full-width, half-width, and overlay treatment
- Layout type auto-assigned in a repeating pattern — no manual control per entry
- Images are optional — entries without images degrade gracefully to a text-only layout that still looks polished
- Data model needs new optional `image` field (ship with no images, add later)

### Vertical line connector
- Thin solid accent-blue line (1-2px) running along the left edge
- Small filled dot markers (4-6px) on the line at each entry position
- Dots are same accent blue color as the line — unified
- Dots appear (fade/scale in) at the same time as their entry content — not static

### Scroll animation
- Entries fade + slide up as they enter the viewport (ranomi.nl pattern: add class on intersection)
- Use Motion `whileInView` or IntersectionObserver for triggering — consistent with existing site patterns
- Reduced motion: respect `MotionConfig reducedMotion="user"` already in App.tsx

### Date presentation
- Large accent-colored year text as the primary visual anchor for each entry (e.g., "2023" in bold accent blue)
- Year extracted from existing date field (e.g., "Sep 2023" → "2023")

### Spacing
- Generous vertical padding per entry (py-16 to py-20 range) for breathing room
- All 8 entries ≈ 3-4 viewport heights total

### Claude's Discretion
- Exact font sizes and weights for year, title, description typography
- Exact image aspect ratios and sizing for each layout variant (large, half, overlay)
- How the overlay layout variant works visually (text over image vs side treatment)
- Fade-in animation timing and easing
- Mobile responsive behavior (stacking, image sizing)
- How text-only entries (no image) are laid out in each variant slot

</decisions>

<specifics>
## Specific Ideas

- "I want the timeline to be inspired by the timeline on this website: https://ranomi.nl/highlights"
- ranomi.nl uses varied image sizes (`-large`, `-halfes`, `-overlay`) to create editorial rhythm — no two adjacent entries look the same
- Multiple text boxes of different sizes filling up the page, with images too
- Current timeline "feels dated" — the small-nodes-on-a-line approach doesn't match the premium feel of the rebuilt Expertise tabs and Project carousel

</specifics>
