  **Step 1 — Create `src/admin/editors/shared/ItemList.tsx`:**
  Compact item picker for list-type editors. Should feel like VS Code's file explorer per user specific idea.
  - Generic props: `{ items: T[], activeIndex: number, onSelect: (index: number) => void, getLabel: (item: T) => string, onAdd: () => void }`
  - Renders a compact vertical list. Each item is a clickable row showing the label.
  - Active item highlighted with `bg-accent/10 text-accent` (using the project's accent color).
  - Non-active items: `hover:bg-gray-50 text-gray-700`.
  - Compact: `text-sm py-1.5 px-3` padding. No icons, just text labels.
  - "+ Add new" button at the bottom: `text-sm text-accent hover:text-accent/80` with lucide `Plus` icon.
  - When list is empty, show a subtle "No items yet" message.

  **Step 2 — Create `src/admin/editors/SkillsEditor.tsx`:**
  List-type editor for SkillGroup[]:
  - Uses `useContentEditor<SkillGroup[]>({ contentType: 'skills', schema: z.array(skillGroupSchema), onDirtyChange, saveRef })`
  - Local `activeIndex` state (default 0).
  - Renders ItemList at top with `getLabel={(item) => item.domain || 'Untitled'}`.
  - Below ItemList: form for the active item.
  - `<SectionHeader>Details</SectionHeader>` then FormField for `domain`.
  - `<SectionHeader>Skills</SectionHeader>` then TagInput for `skills` array.
  - Item field updates: use `setData` to clone the array, update the active item's field, return new array. Call `onDirtyChange(true)`.
  - Add new item: append `{ domain: '', skills: [] }` to the array, set activeIndex to new last index.
  - Delete button at bottom of item form: red text "Delete Group" button. On click, show `window.confirm('Delete this skill group?')`. If confirmed: remove from array, adjust activeIndex, call `onDirtyChange(true)`.

  **Step 3 — Create `src/admin/editors/ToolingEditor.tsx`:**
  List-type editor for ToolingGroup[]:
  - Same pattern as SkillsEditor but with `category` field (instead of `domain`) and `items` TagInput (instead of `skills`).
  - Uses `toolingGroupSchema`, `contentType: 'tooling'`.
  - `getLabel={(item) => item.category || 'Untitled'}`
  - Delete prompt: "Delete this tooling group?"

  **Step 4 — Create `src/admin/editors/TimelineEditor.tsx`:**
  List-type editor for TimelineMilestone[]:
  - Uses `timelineMilestoneSchema`, `contentType: 'timeline'`.
  - `getLabel={(item) => item.title || 'Untitled'}`
  - Form fields: `<SectionHeader>Details</SectionHeader>` then FormField for `date`, `title`, `description` (description uses `multiline`).
  - No TagInput or StructuredArrayField needed — all fields are simple strings.
  - Delete prompt: "Delete this milestone?"

  **Step 5 — Create `src/admin/editors/CourseworkEditor.tsx`:**
  List-type editor for Course[]:
  - Uses `courseSchema`, `contentType: 'coursework'`.
  - `getLabel={(item) => item.code ? \`${item.code} — ${item.name}\` : item.name || 'Untitled'}`
  - Form fields: `<SectionHeader>Details</SectionHeader>` then FormField for `code`, `name`, `descriptor`.
  - Delete prompt: "Delete this course?"

  **Common pattern for all 4 editors:**
  - Props: `{ onDirtyChange: (dirty: boolean) => void, saveRef: React.MutableRefObject<(() => Promise<boolean>) | null> }`
  - Show loading skeleton while `loading` is true.
  - When `data` is null or empty after loading, show only the ItemList (which shows "No items yet" + Add button).
  - All data reads via `useContentEditor` hook (fetches from API). NEVER import from `src/data/*.ts`.
  - Pass relevant `fieldErrors` to form fields. For array items, access errors by index if needed (e.g., `fieldErrors[activeIndex]` or handle top-level array errors).
