In `src/admin/useContentEditor.ts`, replace the `z.flattenError()` call in the `save` function with a custom flattener that produces dotted-path keys. The current code:

```typescript
const flattened = z.flattenError(result.error);
setFieldErrors(flattened.fieldErrors as FieldErrors);
```

Replace with a manual loop over `result.error.issues`:

```typescript
const errors: FieldErrors = {};
for (const issue of result.error.issues) {
  const key = issue.path.join('.');
  if (!errors[key]) errors[key] = [];
  errors[key]!.push(issue.message);
}
setFieldErrors(errors);
```

This produces keys like `'0.title'`, `'0.date'`, `'name'`, `'subtitle'` -- matching what editors look up via `fieldErrors[\`${activeIndex}.field\`]` for array editors and `fieldErrors['field']` for singleton editors.
