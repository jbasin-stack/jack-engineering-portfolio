  <name>Task 1: Build UploadZone component with drag-drop UX and all visual states</name>
  <files>src/admin/UploadZone.tsx</files>
  <action>
Create `src/admin/UploadZone.tsx` -- a reusable inline drop zone component.

Props interface:
```typescript
interface UploadZoneProps {
  /** Accepted file extensions, e.g. ['.jpg', '.jpeg', '.png', '.svg', '.webp'] */
  accept: string[];
  /** Max file size in bytes */
  maxSize: number;
  /** Upload context for server-side routing */
  context: UploadContext;
  /** Current file path (if one exists) */
  currentFile?: string;
  /** Label for the field, e.g. "Project Thumbnail" */
  label: string;
  /** Callback after successful upload with the new file path */
  onUploaded?: (path: string) => void;
}
```

State management:
- `dragState`: 'idle' | 'valid' | 'invalid' -- drives visual appearance
- `dragCounter` ref (number): increment on dragenter, decrement on dragleave, reset to 0 on drop -- prevents flicker from child element boundary crossings (Research pitfall 6)
- `isUploading` (boolean): shows spinner/progress during upload
- `uploadedPath` (string | null): briefly shows the newly uploaded file before resetting to currentFile
- `showCheckmark` (boolean): briefly true after successful upload for green checkmark animation
- `inputRef`: ref to hidden file input for click-to-browse

Drag event handlers:
- `handleDragEnter`: preventDefault, increment dragCounter, check DataTransfer items for valid MIME types (image/* for images, application/pdf for PDFs). Set dragState to 'valid' or 'invalid' based on type check.
- `handleDragOver`: preventDefault only (required for drop to work)
- `handleDragLeave`: preventDefault, decrement dragCounter, if counter reaches 0 set dragState to 'idle'
- `handleDrop`: preventDefault, reset dragCounter to 0, set dragState to 'idle'. Get first file from dataTransfer.files. Client-side validate with `validateUpload`. If invalid, show `toast.error(errorMessage)`. If valid, call `handleUpload(file)`.

Upload handler (`handleUpload`):
- Set `isUploading = true`
- Call `uploadFile(file, context)` from upload.ts
- On success: set `uploadedPath = result.path`, set `showCheckmark = true`, call `toast.success('File uploaded')`, call `onUploaded?.(result.path)`. After 2 seconds, set `showCheckmark = false`.
- On error: `toast.error(result.error || 'Upload failed')`
- Finally: set `isUploading = false`

Click handler: trigger `inputRef.current?.click()` when the zone is clicked

Hidden input: `<input ref={inputRef} type="file" accept={accept.join(',')} hidden onChange={handleInputChange} />`
- `handleInputChange`: get first file, validate, upload (same as drop)

Visual rendering (4 states):

**Idle state (no file):**
- Dashed border (border-dashed border-2 border-gray-300), rounded-lg
- Upload icon (lucide-react `Upload`) centered, gray text "Drop file here or click to browse"
- Label text above the zone

**Idle state (has file):**
- Same dashed border but border-gray-200
- If image: show thumbnail preview (img tag with currentFile or uploadedPath as src, object-cover, max-h-24)
- If PDF: show FileText icon from lucide-react + filename
- "Drop to replace" hint text below preview

**Valid drag-over:**
- Solid accent border (border-solid border-2 border-accent), bg-accent/5
- Scale up to 102%: use `transform: scale(1.02)` with transition
- Text changes to "Drop to upload!"
- Upload icon animates (slight bounce or pulse)

**Invalid drag-over:**
- Solid red border (border-solid border-2 border-red-500), bg-red-50
- Text: rejection reason, e.g. "File type not allowed"
- X icon from lucide-react

**Uploading:**
- Border returns to normal
- Spinner (use lucide-react `Loader2` with `animate-spin`) centered over the preview area
- Semi-transparent overlay

**Success (brief):**
- Green checkmark (lucide-react `Check`) with motion.div scale-in animation (initial scale 0, animate scale 1, with spring)
- Fades out after 2 seconds

Use Tailwind for all styling. Use `motion/react` for the checkmark animation and the scale transition on drag-over. Keep the component self-contained -- it should work as a drop-in for any file field in Phase 10 editors.

IMPORTANT: Detect whether the current file is an image or a document by checking the extension. Images show thumbnails, documents show icon + name.
