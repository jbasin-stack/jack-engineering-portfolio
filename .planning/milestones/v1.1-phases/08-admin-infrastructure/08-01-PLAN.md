2. Create `src/admin/codegen.ts` with two exported functions:

   **`generateDataFile(typeName, exportName, data, isArray)`** - Produces a raw TypeScript source string:
   - Line 1: `import type { ${typeName} } from '../types/data';`
   - Line 2: empty
   - Line 3: `export const ${exportName}: ${typeAnnotation} = ${JSON.stringify(data, null, 2)};`
   - Line 4: empty trailing newline
   - `typeAnnotation` is `${typeName}[]` when `isArray` is true, otherwise just `typeName`

   **`formatAndValidate(rawSource)`** - Formats and validates the generated source:
   - Call `prettier.format(rawSource, { parser: 'typescript', singleQuote: true, trailingComma: 'all', printWidth: 100 })` — this is async
   - Then validate with `ts.createSourceFile('generated.ts', formatted, ts.ScriptTarget.Latest, true)`
   - Check `sourceFile.parseDiagnostics` — if any errors, throw an Error with line numbers and messages using `ts.getLineAndCharacterOfPosition` and `ts.flattenDiagnosticMessageText`
   - Return the formatted string on success

   Use `import * as prettier from 'prettier';` and `import ts from 'typescript';` (both are devDependencies available server-side only).

   The file must use single quotes and follow the project's TypeScript conventions (`import type` for type-only imports is not needed here since prettier and ts are value imports used at runtime in the Vite plugin server context).
  </action>

2. **Create `vite-plugin-admin-api.ts`** at project root (alongside vite.config.ts):

   Export a function `adminApiPlugin()` returning a `Plugin` object with:

   **`name: 'admin-api'`**
   **`apply: 'serve'`** — plugin is completely inert during `vite build`

   **Content Registry** — `CONTENT_REGISTRY` object mapping 9 content type keys to their metadata:
   ```
   hero:       { file: 'hero.ts',       type: 'HeroData',          export: 'heroData',      isArray: false }
   projects:   { file: 'projects.ts',   type: 'Project',           export: 'projects',      isArray: true  }
   papers:     { file: 'papers.ts',     type: 'Paper',             export: 'papers',        isArray: true  }
   skills:     { file: 'skills.ts',     type: 'SkillGroup',        export: 'skillGroups',   isArray: true  }
   tooling:    { file: 'tooling.ts',    type: 'ToolingGroup',      export: 'toolingGroups', isArray: true  }
   timeline:   { file: 'timeline.ts',   type: 'TimelineMilestone', export: 'milestones',    isArray: true  }
   contact:    { file: 'contact.ts',    type: 'ContactData',       export: 'contactData',   isArray: false }
   navigation: { file: 'navigation.ts', type: 'NavItem',           export: 'navItems',      isArray: true  }
   coursework: { file: 'coursework.ts', type: 'Course',            export: 'courses',       isArray: true  }
   ```

   **`configureServer(server)` hook:**

   a. **Terminal message** — Override `server.printUrls` to append an `Admin:` line matching Vite's native style:
      ```
        Admin:   http://localhost:5173/?admin
      ```
      Use cyan ANSI color (`\x1b[36m`) for "Admin:" to match Vite's "Local:" styling.

   b. **Return middleware function** (must return `() => { server.middlewares.use(...) }` so middleware mounts AFTER Vite internals):
      - Match `req.url` starting with `/__admin-api/content/`
      - Extract content type from URL path segment after `/content/`
      - Validate content type exists in `CONTENT_REGISTRY`, return 404 JSON `{ error: 'Unknown content type' }` if not
      - **GET handler**: Use `server.ssrLoadModule('/src/data/' + entry.file)` to read current data, return `mod[entry.export]` as JSON. Add `// TODO: migrate to moduleRunner.import() when ssrLoadModule is removed` comment.
      - **POST handler**: Parse JSON body manually (Buffer collect from `req.on('data')` + `JSON.parse`), call `generateDataFile` + `formatAndValidate` from codegen, then `atomicWrite` the result. Return `{ success: true }` on success, `{ error: detail }` on failure with 400 status.
      - Return 405 JSON `{ error: 'Method not allowed' }` for non-GET/POST
      - Set `Content-Type: application/json` on all responses
      - Wrap entire handler in try/catch, log errors with `server.config.logger.error('[admin-api] ERROR: ...')` (errors only per user decision — no success logging)
      - **Also handle** `/__admin-api/content` (no trailing type) — return JSON list of available content types: `{ types: Object.keys(CONTENT_REGISTRY) }`

   c. **Per-file write queue** — Maintain a `Map<string, Promise<void>>` of pending writes. Before writing, chain onto any existing promise for that file path. This serializes concurrent writes to the same file (last-write-wins).

   d. **Atomic write function** — `atomicWrite(filePath, content)`:
      - Write to `filePath + '.tmp'`
      - Add `filePath` to `activeWrites` Set
      - Rename `.tmp` to target path
      - Retry rename up to 3 times with 100ms delay on `EPERM` or `EBUSY` (Windows antivirus file locking)
      - In `finally`, use `setTimeout(() => activeWrites.delete(filePath), 200)` to clear write lock after chokidar picks up the change

   **`handleHotUpdate({ file })` hook:**
   - Resolve `file` to absolute path
   - If the resolved path is in `activeWrites` Set, return `[]` (empty array suppresses HMR)
   - Otherwise return `undefined` (default HMR behavior)

3. **Update `tsconfig.node.json`** — Change `"include"` from `["vite.config.ts"]` to `["vite.config.ts", "vite-plugin-admin-api.ts"]` so TypeScript checks the plugin file.

4. **Update `vite.config.ts`** — Add import and register the plugin:

