  <action>
1. **Create `src/admin/AdminShell.tsx`** as a minimal stub component:
   - Default export a functional component that renders a simple div
   - Content: a fixed-position left panel placeholder with text "Admin Panel (Phase 9)" and a close mechanism
   - This is a STUB — Phase 9 builds the real UI. This just proves the code boundary works.
   - The stub should:
     - Accept no props (the real AdminShell will manage its own state)
     - Render a `<div>` with basic Tailwind styling: `fixed left-0 top-0 h-screen w-[320px] bg-white border-r border-gray-200 z-[70] p-6 shadow-lg`
     - Show text: "Admin Panel" as an h2 and "Coming in Phase 9" as a paragraph
     - Include a close button that removes `?admin` from the URL and dispatches a custom event or calls `window.history.replaceState` to remove the query param
   - Keep it minimal — no imports from `src/data/` (prevents HMR coupling)

2. **Modify `src/App.tsx`** to add the dev-mode entry gate:

   a. Add `lazy` and `Suspense` imports from React:
      ```typescript
      import { lazy, Suspense } from 'react';
      ```

   b. Add the conditional lazy import BEFORE the App function, at module scope:
      ```typescript
      const AdminShell = import.meta.env.DEV
        ? lazy(() => import('./admin/AdminShell'))
        : null;
      ```
      This pattern ensures: in dev mode, AdminShell is lazily loaded; in production, the entire `import()` expression is dead code because `import.meta.env.DEV` is replaced with `false` at build time, and the ternary resolves to `null`.

   c. Inside the App component, add admin rendering logic:
      - Check for `?admin` query parameter: `const isAdmin = import.meta.env.DEV && new URLSearchParams(window.location.search).has('admin');`
      - After the `<Navigation />` component, conditionally render:
        ```tsx
        {AdminShell && isAdmin && (
          <Suspense fallback={null}>
            <AdminShell />
          </Suspense>
        )}
        ```
      - The `AdminShell && ` guard ensures TypeScript knows it's not null in production.

   d. Do NOT add keyboard shortcut handling here — that belongs in Phase 9 with the full admin shell. The `?admin` URL entry point is sufficient for Phase 8 infrastructure verification.

   e. Preserve all existing imports and JSX structure exactly as-is. The admin gate is purely additive.
  </action>
