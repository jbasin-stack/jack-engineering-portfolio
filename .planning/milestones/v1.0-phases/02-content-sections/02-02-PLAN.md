  <action>
    Create the `src/components/sections/` directory if it does not exist.

    **Skills.tsx:**
    Build a section component that renders `skillGroups` from the data file in a responsive 2x2 column grid per CONTEXT.md locked decision.

    Structure:
    ```
    <motion.section id="skills" aria-label="Technical Skills">
      <div className="mx-auto max-w-5xl">  <!-- consistent container width -->
        <motion.h2>Skills</motion.h2>
        <div className="grid grid-cols-1 gap-12 sm:grid-cols-2 lg:grid-cols-4">
          {skillGroups.map(group => (
            <motion.div role="group" aria-label={`${group.domain} skills`}>
              <h3>{group.domain}</h3>  <!-- natural case bold -->
              <ul>
                {group.skills.map(skill => <li>{skill}</li>)}
              </ul>
            </motion.div>
          ))}
        </div>
      </div>
    </motion.section>
    ```

    Key implementation details:
    - Import `motion, type Variants` from `motion/react`
    - Import `sectionVariants` and `fadeUpVariant` from `../../styles/motion`
    - Import `skillGroups` from `../../data/skills`
    - Wrap `<motion.section>` with `variants={sectionVariants}`, `initial="hidden"`, `whileInView="visible"`, `viewport={{ once: true, amount: 0.15 }}`
    - Apply `variants={fadeUpVariant}` to h2 and each grid column div
    - Section padding: `className="px-6 py-24"` (generous whitespace, NO min-h-screen)
    - Heading: `className="text-2xl font-bold text-ink"` — natural case, not uppercase
    - Grid gap between heading and grid: `mt-12`
    - Domain headings: `className="text-lg font-semibold text-ink"` — natural case bold per CONTEXT
    - Skill items: `className="text-silicon-600"` — name only, no secondary info
    - List: `<ul className="mt-4 space-y-2">` with `<li>` elements
    - NO borders, NO dividers, NO hover effects on items — whitespace-only separation per CONTEXT
    - Semantic HTML: `<section>`, `<h2>`, `<h3>`, `<ul>`, `<li>`, `role="group"`, `aria-label` for AI scraper readability (SKIL-04)
    - Mobile collapse: grid-cols-1 on small, sm:grid-cols-2 on medium, lg:grid-cols-4 on large

    **Tooling.tsx:**
    Build an identical-pattern section component that renders `toolingGroups`. Same visual DNA as Skills per CONTEXT.md locked decision ("Same column grid pattern as Skills").

    Structure mirrors Skills exactly except:
    - `id="tooling"`, `aria-label="Lab and Tooling Proficiency"`
    - Import `toolingGroups` from `../../data/tooling`
    - Heading text: "Lab & Tooling"
    - Grid uses `lg:grid-cols-3` (3 categories vs 4 domains)
    - Maps `group.category` for heading, `group.items` for list
    - `role="group"` with `aria-label={`${group.category}`}`

    Do NOT create any borders, background alternation, or dividers between or within these sections. Whitespace is the primary organizing principle.
