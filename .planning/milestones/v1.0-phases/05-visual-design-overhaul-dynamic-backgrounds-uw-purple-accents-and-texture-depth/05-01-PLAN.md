**AuroraBackground.tsx** (CSS-only aurora gradient wrapper):
- Props: `children: ReactNode`, `className?: string`, `showRadialGradient?: boolean` (default true)
- Renders a relative container with:
  - An absolute-positioned inner div using repeating-linear-gradient with UW purple tones (--color-uw-purple-light, --color-uw-purple-soft, --color-uw-purple-faint)
  - CSS classes: `animate-aurora`, `filter blur-[40px]`, pointer-events-none, opacity-40
  - After pseudo-element with mix-blend-difference for depth
  - Radial gradient mask (ellipse at top center) when showRadialGradient is true
  - Will-change: filter on the blur div for GPU compositing
- Children rendered in a `relative z-10` wrapper
- See RESEARCH.md "Aurora Background Adaptation" code example for exact implementation

**Particles.tsx** (canvas-based particles with mouse magnetism):
- Props: `className?: string`, `quantity?: number` (default 60), `color?: string` (default "#7c5eb5"), `staticity?: number` (default 80), `ease?: number` (default 60), `size?: number` (default 0.4), `vx?: number`, `vy?: number`
- Canvas-based with requestAnimationFrame loop
- Mouse magnetism: particles drift toward/away from cursor based on staticity value
- DPR-aware canvas scaling (window.devicePixelRatio)
- Cleanup: cancelAnimationFrame on unmount, resize observer with debounce
- prefers-reduced-motion: return null (no particles at all)
- Mobile optimization: if window.innerWidth < 768, halve the quantity
- Canvas element: aria-hidden="true", pointer-events-none
- Adapt from magicui Particles source (see RESEARCH.md), change framer-motion imports to motion/react

**NoisyBackground.tsx** (SVG noise texture overlay — lightweight, no canvas):
- Props: `children: ReactNode`, `className?: string`, `gradientFrom?: string` (default "var(--color-uw-purple-faint)"), `gradientTo?: string` (default "var(--color-cleanroom)"), `noiseOpacity?: number` (default 0.3)
- Uses SVG feTurbulence filter for grain texture (NOT canvas — per research recommendation for zero-JS static noise)
- Renders: gradient background div (absolute) + SVG noise overlay (absolute, pointer-events-none, aria-hidden) + children in relative z-10 wrapper
- Use React `useId()` hook to generate unique SVG filter IDs per instance (avoids ID collision when multiple NoisyBackground instances exist on same page)
- See RESEARCH.md "Noisy Gradient Background -- Simplified CSS/SVG Alternative" for exact pattern

**CardSpotlight.tsx** (mouse-tracked radial gradient spotlight):
- Props: `children: ReactNode`, `className?: string`, `radius?: number` (default 300), `color?: string` (default "rgba(75, 46, 131, 0.12)")
- Uses useMotionValue and useMotionTemplate from motion/react for 60fps mouse tracking without React re-renders
- On mouse move: update x/y motion values to cursor position relative to element
- Renders an overlay div with radial-gradient background using motion template
- Overlay is absolute, pointer-events-none, opacity transitions on hover (0 -> 1)
- Must be usable as an INNER wrapper (does not add position or sizing that would interfere with parent layout animations)
- prefers-reduced-motion: render children without the spotlight overlay

**AnimatedGridPattern.tsx** (SVG grid with motion.rect animations):
- Props: `width?: number` (default 40), `height?: number` (default 40), `numSquares?: number` (default 30), `maxOpacity?: number` (default 0.15), `duration?: number` (default 4), `className?: string`
- SVG-based pattern with motion.rect elements that fade in/out at random positions
- Uses ResizeObserver to match SVG viewBox to container dimensions
- Each square: random position, motion.animate with repeat/repeatType for continuous fade cycle
- prefers-reduced-motion: render static grid (no motion animation)
- Adapt from magicui AnimatedGridPattern source, change framer-motion imports to motion/react
